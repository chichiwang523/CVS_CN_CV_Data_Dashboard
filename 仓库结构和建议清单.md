# 仓库结构和建议清单

## 1. 目标

你当前项目适合采用“代码进 GitHub，数据进对象存储”的方式：
- GitHub 负责版本管理、协作、发布
- 原始数据和大体量缓存文件放对象存储（OSS/COS/S3/NAS）

这样可以保证仓库轻量、拉取快、团队协作顺畅。

---

## 2. 推荐仓库结构

```text
china-cv-agent/
├── api/                      # FastAPI 接口
├── dashboard/                # Streamlit 看板
├── scripts/                  # 构建缓存/导出/质量检查脚本
├── src/                      # 核心清洗与分析逻辑
│
├── docs/                     # 文档（建议新增）
│   ├── DEPLOYMENT.md         # 部署手册
│   ├── DATA_DICTIONARY.md    # 字段字典/口径定义
│   ├── RUNBOOK.md            # 运维操作手册
│   └── CHANGELOG.md          # 版本变更记录
│
├── configs/                  # 配置模板（建议新增）
│   ├── env.example           # 环境变量模板
│   └── storage.example.yaml  # 对象存储配置模板
│
├── data_sample/              # 小样例数据（建议新增，< 5MB）
│   └── vehicle_batch_388_sample.parquet
│
├── tests/                    # 单元测试/集成测试（建议逐步补）
│
├── .gitignore
├── README.md
├── requirements.txt
└── 仓库结构和建议清单.md
```

说明：
- `CVdata/`、`cache/`、`output/` 不建议纳入 GitHub 跟踪。
- 如果需要保留目录结构，可仅保留空目录 + `.gitkeep`。

---

## 3. 哪些内容应该上传 GitHub

建议上传：
- 全部代码：`src/`, `dashboard/`, `api/`, `scripts/`
- 文档与配置模板：`README.md`, `docs/`, `configs/*.example`
- 小样例数据（用于 demo / CI 快速验证）

不建议上传：
- 原始数据：`CVdata/`（3.4GB）
- 缓存文件：`cache/*.parquet`
- 导出文件：`output/*.xlsx`, `output/charts/*.png`
- 大型日志、临时文件、环境目录

## 5. 数据存储建议（生产）

推荐数据分层：

```text
raw/      # 原始 Excel/JSON（归档层）
silver/   # 清洗后标准化 Parquet（查询主层）
gold/     # 预聚合统计 Parquet（看板/API 高速层）
```

建议命名规则：
- `raw/vehicle_batch_388.xlsx`
- `silver/all_batches_cleaned.parquet`
- `gold/energy_distribution_by_batch.parquet`
- `gold/abs_supplier_trend.parquet`

## 6. 团队协作规范（最小可用）

分支策略：
- `main`：可发布版本
- `dev`：集成开发分支
- `feature/*`：功能分支（如 `feature/dashboard-free-field`）
- `fix/*`：修复分支

提交信息建议：
- `feat: 新增电驱拓扑说明模块`
- `fix: 修复制动系统页面列名错误`
- `docs: 更新部署说明`

发布建议：
- 用 tag 标记版本：`v1.0.0`, `v1.1.0`
- 每次发布维护 `CHANGELOG.md`

---

## 7. 上线部署最小流程（建议）

1. 代码拉取
```bash
git clone <repo_url>
cd china-cv-agent
pip install -r requirements.txt
```

2. 同步数据（从对象存储拉取）
```bash
python scripts/sync_data.py
```

3. 构建缓存
```bash
python scripts/build_cache.py
```

4. 启动服务
```bash
streamlit run dashboard/app.py --server.port 8501
uvicorn api.main:app --host 0.0.0.0 --port 8000
```
